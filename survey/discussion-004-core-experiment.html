D1. TermiGen critic 공정성 → 엔진 재현 말고 통제된 A

비교 변수 = 데이터 provenance지 TermiGen 엔진 아님. 우리 gpt_agent.py가 이미 TermiGen-style 자체 재구현(sft_all.jsonl=A) → A/B/C가 단일 harness에서 돔. critic은 A 내부 주입 품질관리일 뿐; 공정성은 모든 arm 동일 downstream(base·양·recipe·eval·최종 verified 필터)로 확보.

D2. DAgger(B) 구현 → C와 같은 실패 prefix, 작성자만 teacher

B·C가 동일 student 실패 prefix(ECE) 공유. C=student가 grounded 힌트로 복구(힌트 strip), B=teacher가 그 prefix 이어받아 복구. 둘 다 reward 0→1만. 입력 동일·작성자만 다름 = "작성자" 변수 완벽 격리. 인프라 있음(recovery_agent resume + gpt_agent teacher).

D3. 통일 세팅

student=Qwen3.5-4B 고정 · teacher=하나 고정 · task=funnel 정제(flaky/verifier-bug/timeout 제외 + teacher가 best-of-N으로 푼 solvable만) · base corpus 고정+recovery slice만 swap+양 매칭 · eval=TerminalBench + hint-free 복구. arms는 recovery provenance에서만 다름.

D4. 저yield 복구 절차

단발 힌트 복구 1/7(실측), teacher 못 푸는 task엔 무의미 → grounded 전제. recovery 작성자 = teacher-개입 스펙트럼: B(전부)→C-iterative(grounded 힌트 k회)→C-single→rerun(0). 제안: C=grounded 힌트 최대 k≤3 iterative + 힌트 strip, yield·k 보고. 지표 = "hint-strip된 C로 학습한 student가 hint-free 테스트에서 B만큼/이상". k도 ablation 축.
